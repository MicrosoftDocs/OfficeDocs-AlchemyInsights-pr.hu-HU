# <a name="troubleshoot-quarantine-state"></a>Karantén állapotának elhárítása

Az Azure Active Directory (AD) kiépítési szolgáltatása figyeli a konfiguráció állapotát. Ezenkívül a nem egészséges appokat karanténállapotba **helyezi.** Ha a célrendszeren végzett hívások nagy része vagy egésze folyamatosan sikertelen, akkor a kiépítési feladat karanténban **van megjelölve.** Hiba lehet például az érvénytelen rendszergazdai hitelesítő adatok miatt **kapott hiba.** További információ: Az alkalmazás [kiépítése karanténállapotban.](https://docs.microsoft.com/azure/active-directory/app-provisioning/application-provisioning-quarantine-status)

A felhőszinkronizálással kapcsolatos hibák elhárításáról a [karanténba helyezett problémák kiépítése témakörben kaphat további információért.](https://docs.microsoft.com/azure/active-directory/cloud-sync/how-to-troubleshoot#provisioning-quarantined-problems)
